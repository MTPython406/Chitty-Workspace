# ChittyWorkspace
**Stack:** Rust + Python sidecar
**Entry:** src/main.rs
**Build:** `cargo build` (touch src/server.rs after editing assets/chat.html)

## Key Files
- src/main.rs — CLI entry point, spawns server + UI
- src/server.rs — Axum HTTP server, all API endpoints, SSE chat streaming
- src/providers/mod.rs — Provider trait, ProviderId enum (Openai, Anthropic, Google, Xai, Local)
- src/providers/local_sidecar.rs — Local GGUF model provider via Python sidecar
- src/providers/cloud.rs — Anthropic cloud provider
- src/providers/adaptors/ — OpenAI-compatible SSE parser, xAI provider
- src/config/mod.rs — App config (config.toml), LocalModelConfig, API key storage
- src/gpu.rs — GPU detection (nvidia-smi), VRAM estimation
- src/huggingface.rs — Sidecar client (inference, media, training, STT)
- src/chat/mod.rs — Chat engine, system prompt, context assembly
- src/agents/mod.rs — Agent CRUD, sub-agents
- src/skills/mod.rs — Skills system (SKILL.md parser)
- src/tools/ — Native tools (file, terminal, web, media, browser)
- src/media/mod.rs — MediaAdaptor trait (image, video, TTS, STT)
- src/storage/ — SQLite database, schema migrations
- assets/chat.html — Full frontend (embedded at compile time)
- sidecar/inference_server.py — Python sidecar (GGUF inference, media, training, STT)
- sidecar/media_engine.py — Image/video/TTS via diffusers
- sidecar/training_engine.py — LoRA/QLoRA fine-tuning

## Local Engine Capabilities
- Text inference: GGUF models via llama.cpp
- Image generation: Flux, SDXL, SD3, Kandinsky, PixArt
- Video generation: CogVideoX, Wan, LTX-Video, AnimateDiff, Mochi
- Text-to-Speech: Bark, XTTS, SpeechT5, Parler
- Speech-to-Text: Whisper (auto-downloads on first use)
- Fine-Tuning: LoRA/QLoRA via peft + trl
